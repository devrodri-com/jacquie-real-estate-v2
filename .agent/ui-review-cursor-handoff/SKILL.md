# UI Review + Cursor Handoff - Jacquie Real Estate

## Goal
Review an existing UI critically, identify the highest-value improvements, and return a precise prompt ready for Cursor.

This site is not a marketplace. It is a **trust-based real estate website** designed to convert visitors into **direct conversation**, mainly through WhatsApp.

## Product truths you must preserve
- The site sells **confidence and relationship**, not inventory first.
- Jacquie is positioned as:
  > "Your trusted person in Miami"
- The business priority is:
  1. Property Management
  2. Listings
  3. Pre-construction
- Home should be:
  - trust-led
  - conversion-oriented
  - secondarily exploratory
- Premium means:
  - clarity
  - restraint
  - strong hierarchy
  - no visual clutter
  - no filler sections

## Use this skill when
- the user shares a section, page, screen, or visual flow,
- the issue is about hierarchy, trust, branding, clarity, or conversion,
- the user wants critique plus a Cursor-ready handoff,
- the requested change should stay MVP-safe.

## Do not use this skill when
- the task is primarily backend/data-flow related,
- the user wants direct implementation,
- the task is a tiny isolated visual tweak,
- the product direction is still undefined.

## Required process
1. Evaluate the UI with a critical product and conversion lens.
2. Separate:
   - what is already working
   - what weakens trust, clarity, or narrative
3. Keep project context in mind:
   - Jacquie is a person-led brand
   - WhatsApp is the main CTA
   - trust must come before exploration
   - sections should not compete equally
4. Recommend only the highest-value changes first.
5. Avoid broad redesign unless explicitly requested.
6. Produce a precise Cursor prompt that executes safely.

## What to look for
- whether the first 5–8 seconds communicate a dominant idea
- whether trust appears early enough
- whether sections compete instead of support narrative
- whether Property Management is given correct priority
- whether Listings are acting as support instead of becoming the whole story
- whether Precon is too heavy for its actual business role
- whether “About” comes too late
- whether cards are too white / too weak / too generic
- whether the page feels like a portal instead of a trust-led landing
- whether the CTA is too diluted
- whether the tone is too generic-corporate instead of personal-premium
- whether the proposed change should stay inline or move into a small focused component/helper

## Jacquie-specific review criteria
Ask:
- Does this screen make Jacquie feel like a trusted person in Miami?
- Does this screen reduce uncertainty for a first-time investor?
- Does this screen create confidence before asking the user to explore?
- Is WhatsApp visually and strategically supported?
- Is the hierarchy aligned with:
  1. Property Management
  2. Listings
  3. Precon
- Is this section helping conversion, or just filling space?

## Output format
Return:
1. what is working
2. what is weakening the screen
3. the highest-priority improvement
4. a short plan
5. a precise Cursor prompt

## Rules
- do not implement changes
- do not edit files
- do not redesign the whole page unless explicitly requested
- keep recommendations aligned with the actual business goal
- prefer MVP-safe, high-impact, low-risk improvements
- avoid vague design feedback
- if suggesting extraction, keep it small and justified
- when preparing the Cursor prompt, use this exact framing:

PROMPT PARA CURSOR
------------------
[instrucciones exactas]

FIN DEL PROMPT

- Cursor must execute, not decide
